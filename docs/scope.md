# Scope — aria 기능 범위

> 상태: DRAFT
> 이 문서는 aria 가 **무엇을 해주는 비서인지** 정의한다. 비범위는 `out-of-scope.md` 참조.

## 제품 정체성
- 이해·정리형 개인 비서. 자율 실행형 에이전트 아님.
- 결과물의 기본값은 "요약/제안/초안" 이며, 실행은 사용자 승인을 거친다.

## 주요 기능 (In-Scope)

### 1. 문서 (Documents)
- 업로드된 문서 읽기
- 요약
- 여러 문서 간 비교
- 특정 관점에서의 분석
- TODO: 지원 포맷 목록 (PDF, DOCX, TXT, Markdown, …)
- TODO: 청크 분할/인덱싱 전략

### 2. 메일 (Mail)
- 메일 읽기 (Gmail 연동, 최소 권한)
- 스레드/메일 요약
- 답장 초안 작성
- 발송은 **이번 범위 밖** (승인 플로우 별도 설계 전까지 비활성)
- TODO: 스코프 목록 정의 (readonly metadata / full read)

### 3. 캘린더 (Calendar)
- 일정 읽기
- 오늘/이번 주 일정 브리핑
- 일정 제안(슬롯 추천)
- 승인 후 일정 생성
- 삭제/수정은 승인 매트릭스 확정 전까지 비활성

### 4. 메모 (Memos)
- 사용자가 명시한 경우에만 메모 생성
- 메모 읽기, 검색, 요약
- 자동 기억/자동 사실 추출 없음

### 5. 웹 검색 / 날씨 (보조)
- 웹 검색 결과 요약 (비신뢰 입력으로 처리)
- 현재/예보 날씨 조회
- 둘 다 보조 기능이며, 핵심 UX가 아님

## 명시적 가정
- 사용자 1인 기준 설계부터 시작 (RLS는 multi-tenant 호환 구조).
- 한국어 중심 UX. 다국어는 나중 단계.
