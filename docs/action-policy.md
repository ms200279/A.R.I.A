# Action Policy — 액션 승인 매트릭스

> 상태: DRAFT
> 모든 쓰기 계열 액션은 이 문서의 분류에 따라 처리된다.
> 이 문서와 `lib/policies` 구현은 1:1 로 동기화되어야 한다.

## 액션 등급

| 등급 | 설명 | 승인 | 예시 |
|---|---|---|---|
| **Read** | 조회/검색. 상태를 바꾸지 않음 | 불필요 | 문서/메일/일정/메모 읽기, 웹 검색, 날씨 |
| **Suggest** | 결과물 생성만. 저장 안 됨 | 불필요 | 요약, 비교, 분석, 답장 초안, 일정 제안 |
| **Create (low-risk)** | 소유자 로컬 자원 생성 | 경량 확인 (UI 버튼) | 메모 저장, 초안 저장 |
| **Create (approval)** | 외부 시스템 변경 | 명시적 승인 플로우 | 캘린더 이벤트 생성 |
| **Sensitive** | 기본 비활성. 별도 설계 필요 | 강한 승인 | 메일 발송, 삭제, 외부 공유, 웹 조작 |

## 승인 플로우 개요 (초안)
1. LLM/오케스트레이터가 **pending_action** 레코드를 만든다.
2. 사용자에게 요약 카드(무엇을, 어디에, 왜)를 보여준다.
3. 사용자가 승인하면 `app/api/approvals/...` 가 실제 실행을 트리거한다.
4. 실행 결과는 `execution_log` 에 기록한다.
5. 초안과 pending_action, execution_log 는 서로 분리된 테이블이다.

## 정책 체크 의무

## 기본값
- 기본값은 항상 **더 보수적인 쪽**.
- 애매하면 Suggest 로 떨어뜨리고 실행은 사용자에게 맡긴다.
